1. Introduction

The Job Portal is a full-stack web application designed to bridge the gap between job seekers and employers. It provides a comprehensive platform for posting job listings, submitting applications, tracking application status, communicating in real-time, managing resumes with ATS scoring, conducting skill assessments through quizzes, and administering the entire recruitment lifecycle.

The system is built using Spring Boot 4.0.4 for the backend REST API and React 19 with Vite for the frontend single-page application. It supports three distinct user roles with role-based access control: Job Seeker, Employer, and Administrator.

Key capabilities include real-time messaging via WebSocket, automated email notifications, JWT-based stateless authentication, and comprehensive analytics dashboards for both employers and administrators.

3. Modules

1. Authentication & Authorization

Registration, login, JWT tokens, role-based access control (JOBSEEKER, EMPLOYER, ADMIN)

2. Job Management

Create, edit, delete, search, and filter job listings by title, location, type, salary, and experience

3. Application Management

Apply to jobs, track status through 8-stage pipeline, employer review with ratings and feedback

4. Resume Management

Upload multiple resumes, ATS scoring, keyword matching with jobs, resume organization

5. Messaging & Communication

Real-time chat between job seekers and employers, unread tracking, conversation history

6. Quiz & Assessment

Employers create skill quizzes, candidates take timed assessments, automated scoring

7. Admin Management

User management (view/edit roles/delete), job management, platform-wide statistics

8. Company Profile

Company branding with logo, banner, description, industry, size, and social links

9. Notification System

Real-time in-app notifications via WebSocket, click-to-navigate, mark as read

10. Job Alerts

User-configurable alert preferences, automatic job matching, email + in-app notifications

11. Saved Jobs

Bookmark and save jobs for later viewing and comparison

12. Analytics & Dashboard

Per-job analytics, employer aggregate metrics, application funnel tracking

5. Use Case Diagram

Job Seeker Use Cases

  • Register and login with email/password
  • Search and filter jobs (title, location, type, salary, experience)
  • View detailed job descriptions and company information
  • Apply to jobs with cover letter and resume selection
  • Track application status through 8-stage pipeline
  • Take skill assessment quizzes for job screening
  • Upload and manage multiple resumes with ATS scoring
  • Save/bookmark jobs for later viewing
  • Chat with employers in real-time
  • Configure job alert preferences
  • View and manage real-time notifications

Employer Use Cases

  • Register and login with email/password
  • Post new job listings with detailed descriptions
  • Edit and manage existing job postings
  • View and filter applications per job
  • Update application status with ratings and feedback
  • Create skill assessment quizzes for candidate screening
  • Chat with candidates in real-time
  • Manage company profile (logo, banner, description)
  • View job analytics (views, applications, status funnel)

Admin Use Cases

  • View all registered users on the platform
  • Change user roles (JOBSEEKER ↔ EMPLOYER ↔ ADMIN)
  • Delete users and their associated data
  • View all job listings across the platform
  • Delete jobs and associated applications
  • View platform-wide statistics (users, jobs, applications)

11. Security Features

Password Encryption

BCrypt hashing via Spring Security PasswordEncoder — passwords never stored in plain text

JWT Authentication

Stateless token-based auth with 24-hour expiration, signed with HS256 algorithm

Role-Based Access

@PreAuthorize annotations enforce role checks on every protected endpoint

CORS Configuration

Whitelisted origins restrict API access to authorized frontend domains only

Input Validation

@Valid annotations and custom validators on all request bodies

SQL Injection Prevention

All queries use JPA parameterized queries — no raw SQL concatenation

XSS Prevention

React auto-escapes all output; no dangerouslySetInnerHTML used

File Upload Validation

PDF-only restriction, file type and size validation before upload
